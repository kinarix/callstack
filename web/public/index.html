CALLSTACK

A brutalist Progressive Web App for API testing with local SQLite storage

Launch CALLSTACK Download README

✨ Features

  • Full REST API testing (GET, POST, PUT, DELETE, PATCH)
  • Local SQLite storage - all data stays in your browser
  • Google Sign-In for multi-device sync (optional)
  • Works completely offline once installed
  • Request builder with params, headers, and body
  • Response viewer with timing and size metrics
  • Auto-save - never lose your work
  • Brutalist dark mode design

🚀 Quick Start

Step 1: Test Locally

You're already here! Just click "Launch CALLSTACK" above to start using it.

For local development, serve these files with any HTTP server:

# Python
python -m http.server 8000

# Node.js
npx http-server -p 8000

Step 2: Deploy (Optional)

Deploy to any static hosting service for HTTPS (required for PWA installation):

  • GitHub Pages: Free, easy, includes HTTPS
  • Netlify: Drag-and-drop deployment
  • Vercel: One-command deployment
  • Cloudflare Pages: Global CDN

Step 3: Enable Google Sign-In (Optional)

To sync requests across devices:

  1. Create a Google Cloud Project
  2. Enable OAuth 2.0
  3. Get your Client ID
  4. Update GOOGLE_CLIENT_ID in callstack.html

See README.md for detailed instructions

⚠️ CORS Notice: Like all browser-based API tools, you may encounter CORS errors when testing certain APIs. This is a browser security feature, not a bug. See the README for workarounds.
✓ Privacy First: All data is stored locally in your browser. Nothing is sent to external servers (except when you make API requests, of course!).

🎯 Use Cases

  • Test REST APIs during development

  • Share API request collections with team
  • Test APIs offline (with saved requests)

  • Alternative to Postman for simple testing

🔧 System Requirements

  • Modern browser (Chrome 67+, Edge 79+, Firefox 60+, Safari 11.1+)
  • JavaScript enabled
  • LocalStorage enabled (not private/incognito mode)
  • HTTPS hosting (for PWA installation and Google Sign-In)

🎨 Design Philosophy

CALLSTACK embraces a brutalist aesthetic with:

  • Dark mode for reduced eye strain during long dev sessions
  • Monospace typography (JetBrains Mono) for code-centric work
  • Geometric precision with clean grids and borders
  • Neon accents for visual hierarchy and method differentiation
  • No bloat - single HTML file, minimal dependencies
